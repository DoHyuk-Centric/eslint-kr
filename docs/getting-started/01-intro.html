ESLint란?

ESLint는 JavaScript 코드에서 문제를 찾아내고 수정하는 정적 분석 도구입니다. 이 문서에서는 ESLint가 무엇인지, 왜 필요한지, 그리고 비슷한 도구들과 어떻게 다른지 설명합니다.

ESLint란 무엇인가요?

ESLint는 코드를 실행하지 않고도 문제를 찾아내는 정적 분석 도구(linter)입니다. 코드를 저장하는 순간 오류, 잠재적 버그, 스타일 위반을 즉시 알려줍니다.

예를 들어 아래 코드를 작성했다고 가정해 봅시다.

JavaScript 
// ❌ ESLint 없이 — 실행 전까지 문제를 알 수 없음
var userName = "홍길동"
var userAge = 25    // 선언만 하고 사용하지 않음

console.log(userName)
JavaScript 
// ✅ ESLint 적용 후 — 저장하는 순간 경고 표시
const userName = "홍길동"  // var → const 자동 수정 가능

// ⚠ 'userAge'는 선언되었으나 사용되지 않음 (no-unused-vars)
const userAge = 25

console.log(userName)
ℹ️
참고 ESLint는 npm에서 주간 5,280만 회 이상 다운로드되는 JavaScript 생태계의 핵심 도구입니다. Microsoft, Airbnb, Netflix, Meta 등의 기업에서 사용합니다.

왜 써야 하나요?

1. 버그를 미리 잡는다

런타임 에러는 사용자가 실제로 그 코드를 실행하기 전까지 발견하기 어렵습니다. ESLint는 코드를 저장하는 즉시 잠재적 문제를 표시해 줍니다.

2. 팀 코드 스타일을 통일한다

팀원마다 서로 다른 스타일로 코드를 작성하면 나중에 읽고 유지하기 어려워집니다. ESLint로 규칙을 한 번 정해두면 누가 작성해도 일관된 스타일이 유지됩니다.

3. 자동으로 수정해 준다

많은 규칙들은 --fix 옵션으로 자동 수정이 가능합니다. VS Code 익스텐션을 함께 사용하면 파일을 저장할 때 자동으로 고쳐줍니다.

ESLint vs Prettier

처음 접할 때 가장 많이 헷갈리는 부분입니다. 두 도구는 역할이 다릅니다.

구분ESLintPrettier
주된 역할 코드 품질 검사 코드 형식(포맷) 통일
잡아내는 것 버그, 나쁜 패턴, 미사용 변수 들여쓰기, 따옴표, 줄바꿈, 세미콜론
예시 no-unused-vars, eqeqeq 탭 크기, 따옴표 종류
자동 수정 일부 가능 거의 모두 가능
✅
권장 사항 실무에서는 ESLint와 Prettier를 함께 사용합니다. 역할이 겹치지 않아 충돌 없이 쓸 수 있습니다. 단, ESLint의 일부 포맷 관련 규칙과 Prettier가 충돌할 수 있으므로 eslint-config-prettier를 함께 설치하는 것이 좋습니다.

이름의 유래

ES는 ECMAScript(JavaScript의 공식 표준 명칭), Lint는 코드에서 잠재적 오류나 스타일 문제를 검사하는 행위를 뜻합니다.

Lint라는 단어는 원래 섬유에 붙은 보푸라기를 제거하는 도구에서 유래했습니다. 1978년 Bell Labs의 Stephen Johnson이 C 언어 코드 검사 도구의 이름으로 처음 사용했습니다.

⚠️
버전 안내 ESLint v9부터 설정 파일 형식이 기존 .eslintrc.*에서 eslint.config.mjs로 변경되었습니다. 이 가이드는 v9 이상의 flat config를 기준으로 설명합니다.
